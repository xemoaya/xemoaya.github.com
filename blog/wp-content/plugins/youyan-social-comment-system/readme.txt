== Description ==

在广大用户的支持下，友言迎来了2.0版本。这一版本里，我们实现了：
1. 修复有的用户遇到的在发布新浪和腾讯文章微博时出错的问题
2. 改变插件界面，采用跨域通信的技术，直接嵌入网站管理后台。以后友言有什么功能升级，基本不必升级插件了。
3. 修复在导入评论时，某些文章（post）或页面（page）不能成功显示的问题


而通用的友言系统，则再一次经历了重大的升级，包括：
1. 全面的评论审核系统。支持发布后审核或是审核后发布。四种审核状态：正常，待审核，垃圾评论，已删除
2. 敏感词过滤系统。包括系统自带与自定义敏感词过滤。
3. 全面的黑名单系统。包括邮箱，IP地址，用户名三种黑名单
4. 全面的热榜插件，包括全站最热，最新评论；全站最热，最新文章插件。每种插件有一系列自定义选项。
5. 站长自定义转发样式。再不是单纯系统默认的样式，站长可以亲手DIY
6. 站长可选是否转发图片，系统更加智能的图片选择
7. 回复数支持分页呈现，当一条评论的回复较多时，不必立刻全部调用，而是可以照样分页调用。
8. 新回复出现的位置可设置
9. 置顶登陆按钮新增一套短款按钮供站长选择。
10. 表情，社区功能站长可选择。登陆后是否隐藏置顶登陆按钮站长可选择。

还有一系列其他优化，包括修正同时打开多个使用友言的网站可能出现的问题，大大加快了加载速度等。


1.9.5 是又一次重要的更新，解决了困扰我们和不少用户的许多问题，请升级


在用户的反馈之下，我们用互联网敏捷开发，迭代更新的方式，迅速的完成了又一个重要的更新：1.9 版本，请用户尽早更新，1.9 一下版本可能出问题

2. 添加置顶的登陆按钮
3. 社交网络反向同步，我们能将同步到微博上的评论再取回来，同步到友言的评论系统中。
4. 以及其他许多增强可用的细节调整




1.8 版本是一个非常重要的版本更新，请用户一定要升级，十几项功能更新详见change log，以前的友言插件将不能正确使用。请用户一定要升级，谢谢！


在美国几乎所有网站的评论系统都采用第三方的社交评论解决方案。主要的提供商是两家：Facebook Comment Box（http://developers.facebook.com/docs/reference/plugins/comments/）和Disqus （http://disqus.com/）。社交评论框的优势是明显的：给网站增加社交属性，通过实名制提高评论质量，为网站吸引更多的社交回流流量，帮助站长更好的得到统计分析的信息。

然而，我在国内看到，少有第三方的解决方案。新浪微博和人人网有一个评论插件，但是他们只支持自己的账号登陆同步。相比之下，Facebook的评论框，还同时支持Yahoo，Hotmail，Twitter等账号登陆，开放的胸怀让人赞叹。因此我决定做一个支持所有主流中文社交网络，用户体验完美，站长自定义功能全面，统计分析功能强大的社交评论插件 ---- 就这样，友言诞生了。它的优势在于：

1. 最好用的社交评论解决方案：部署插件或添加两行代码即可使用   ------   您的网站不再孤单，与社交网络亲密相连。

2. 自定义功能最全面的社交评论插件：框体宽度，最大评论字数，单页评论数，反馈样式，文字。。。乃至界面CSS都能由站长根据需求自由配置 ------ 您的网站，永远不用担心“撞衫”，

3. 管理与统计功能最丰富的社交评论插件：活跃用户，热门文章，每日评论数，回流数量，热门回流。。。。 站长均能轻松管理查看。

4. 支持社交网络最全面的评论插件：支持用 新浪微博，人人网，腾讯微薄，QQ空间，搜狐微博，网易微博，开心网账号登陆，评论自动同步。

5. 同步样式最好的社交评论插件：站长每发布一篇文章，支持自动发布微博（需站长绑定微博账号），文章内的评论，以转发并回复站长的微博的方式同步。

== Installation ==

1.  将文件下载，解压缩到`/wp-content/plugins/`文件夹。子文件夹名为"youyan-social-comment-system"
2. 进入wordpress管理后台，按照提示操作，第一次登陆需要注册站长账号。


== Frequently Asked Questions ==
1. 暂不支持功能？
订阅评论RSS，评论后审阅系统，
。。。（期待您的建议： admin@uyan.cc）

2. 为什么站长需要在网站根目录下放置代理文件（proxy.html）
“友言”通过html的iframe标签嵌入网页中，为了能自动调整框架高度（当用户添加和删除评论时），我们需要站长在网站的根目录下放置这样一个代理文件。（下载链接在友言主页 uyan.cc，或是wordpress的管理界面中）

3. ”友言“参照的服务是什么？
美国的 "Facebook comment box" 和 "Disqus Comment System"

4. “友言”的目标是什么？
社会化评论系统，为所有中文网站提供服务

5. 能否对自定义样式提供技术帮助？
欢迎邮件 help@uyan.cc

= 1.8 =
1.	首先，当然我们修复了所有已知的兼容性问题，现在在所有的浏览器，IE6到9，搜狗，360，Chrome, Firefox, Safari，都能够完全正确的运行。
2.	站长配置登陆选项和顺序。
3.	邮件提醒
4.	全面更新的统计系统。
我们对用户评论和回访做了全面的分析和呈现。现在，您能够从“用户”，“评论”，“社交网络”三个维度检测到自己网站的评论和回访趋势。在您的网站上，某个用户有多少条评论，带来多少回访。每种社交网络有多少条评论，带来多少回访。乃至于具体到每一条评论带来多少回访，都能够精确的呈现。您可以很容易的发现本网站的核心活跃用户。
5.	更好的用户评论管理系统。
我们现在支持以页面为单位进行评论与管理。同时，有的网站有多名编辑，但只有一名站长。站长则可发放某一页面的管理权限，生成一个只有当前页面权限的链接给编辑。
6. 更好的“顶”和“踩”。
支持对本篇文章的“顶”和“踩”，“顶”之后用户可选直接分享文章。对文章和评论的“顶”和“踩”再不需要用户登陆。可以登陆，也可以在未登录时进行（用cookie判断最近进行）
7。 评论社区。，能看每一个用户的最新评论以及活跃社区。帮助用户建立评论记录，帮助站长建立评论社区。
8. 自定义违章标题，图片，等
我们默认的同步信息样式为：
我才在“  <网站名>（<网站域名>）”的页面《<页面标题>》发表了评论: “<用户评论>” http://t.cn/ar2GGH <图片>
其中网站名在站长的管理界面里设置。网站域名自动提取。网站标题为为网页的 “title” 标签，图片默认为页面中第一个<img>标签。
站长可以自己配置想要的标题和图片。只需在网页里加入如下内容：
<meta name= “UYPageTitle” content= “ 您设置的微博转发文章标题” />
<meta name= “UYImg” content= “ 您设置的微博转发图片” />
9. 多页面共享评论框
友言默认取页面URL作为一组评论的标识。如果站长想要多个页面（有不同的URL）共享一个评论框，可以在这些页面里指定：
<meta name= “UYId” content= “ 您设置的页面ID” />
设为相同ID，则可共用一个评论框。
10。(wordpress插件专有)，导入导出数据到友言中。站长可以点击多次导入数据，每次只有新评论（如果有的话）才会被导入（同理，导出也是）
强烈建议站长选择导入现有wordpress数据，然后禁用原有wordpress评论系统，以使界面美观。
11. (wordpress插件专有)，文章发微博。
当站长绑定自己的新浪或者腾讯的微博账号后，在文章发布和更新的界面会分别出现下面的选择框。默认同步，若站长不勾选，则自动用站长账号为文章发微博。
12. (wordpress插件专有)，已有“文章微博”的文章下评论处理。
用户在页面内的评论，将以自动转发并评论站长的文章微博的形式进行。
13. (wordpress插件专有)，配置站长自己的APPKEY
您不光可以用友言的APPKEY，也可以用自己的APPKEY
14. 其他
我们还有十多处各种小的调整或升级，如站长可自定义匿名评论用户的默认URL，更快的加载速度，非常稳定的社交网络分享（迄今位置我们测试数百起，没有失败过一起）。我们的保留特色：十几项站长的自定义属性，和自定义CSS等。
